# TRIZ Core Analysis - Swarm Audit Enablement

**Date:** 2026-04-08
**System:** Swarm-driven repository audit for do EPUB Studio

---

## System Scope

- Multi-agent analysis of `do EPUB Studio` repository
- Perspectives: feature coverage, implementation completeness, documentation, tests, architecture, security
- Output: actionable gaps + remediation plan stored in `analysis/SWARM_ANALYSIS.md`

---

### Contradiction 1

**Improving:** Coverage depth (thoroughness of analysis)
**Worsens:** Throughput (time/effort to inspect entire monorepo)
**Why:** Exhaustive inspection of all packages ensures gap detection but dramatically increases time-to-report, delaying remediation and consuming agent budget.
**Reality:** We assume depth requires scanning every file. In reality, targeted sampling guided by heuristics and metadata can maintain depth on critical paths without linear cost.

**TRIZ Principles Available:**

- Segmentation (split repo by capability and assign focused agents)
- Prior action (preload context via plans/ADRs to narrow scope)
- Dynamicity (adjust sampling strategy based on findings)

---

### Contradiction 2

**Improving:** Automation scale (parallel sub-agents synthesizing observations)
**Worsens:** Signal quality (risk of duplicated/noisy findings)
**Why:** Launching many agents speeds discovery but increases redundant or conflicting reports, diluting confidence and requiring manual de-duplication.
**Reality:** Automation can be structured with explicit role prompts and shared templates to standardize outputs and reduce noise.

**TRIZ Principles Available:**

- Asymmetry (differentiate agent roles by perspective)
- Coordination (introduce shared rubric + dedup step)
- Feedback (loop aggregated findings back for confirmation)

---

### Contradiction 3

**Improving:** Evidence granularity (line-level citations, dependency mapping)
**Worsens:** Cognitive load (analysis summary becomes unwieldy)
**Why:** Including deep references for every issue ensures auditability but overwhelms readers, obscuring priority and relationships.
**Reality:** Layered reporting (headline summary + appendix) preserves details without bloating top-level insights.

**TRIZ Principles Available:**

- Nesting (main findings + appendices)
- Local quality (variable detail per gap severity)
- Skipping (defer low-priority evidence to referenced files)

---

## Next Steps

2. Define swarm plan under `plans/` aligning with resolved strategies.
3. Execute multi-agent review adhering to segmentation + coordination rules.
