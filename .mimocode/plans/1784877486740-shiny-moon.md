## Root Cause Analysis: E2E Failures

All 20 failures are **mobile-only** (`@mobile` tag, iphone + pixel viewports).

**Root cause confirmed**: The reader toolbar uses **container queries** for
responsive layout (`ReaderToolbar.tsx:160` → `cq-reader-toolbar-actions`,
line 295 → `cq-reader-toolbar-overflow`).

- **Desktop**: Direct icon buttons visible in toolbar (Search, Comment, Bookmarks, Settings)
- **Mobile**: Only the "More Options" ⋮ button is visible (`aria-label="More Options"`);
  all actions are in a dropdown menu (lines 315-479)

The E2E tests try to click desktop-style toolbar buttons that are **hidden on
mobile** via container queries. On mobile, the tests must:
1. Click `getByRole('button', { name: 'More Options' })` to open the overflow menu
2. Then click the desired option text from the dropdown (Search, Settings, etc.)

**Failing test files and buttons:**

| File | Button | Error | Fix |
|------|--------|-------|-----|
| `edge-cases.spec.ts:125` | Settings | `element(s) not found` | Use overflow menu on mobile |
| `accessibility-audit.spec.ts:52` | Settings | `element is not visible` | Use overflow menu on mobile |
| `accessibility-audit.spec.ts:81` | Settings | `element is not visible` | Use overflow menu on mobile |
| `in-book-search.spec.ts:36,49,64` | Search | `element(s) not found` | Use overflow menu on mobile |
| `login-and-book-load.spec.ts:117` | Settings | `element(s) not found` | Use overflow menu on mobile |
| `reader-annotations-and-admin.spec.ts:22` | Comment | `element(s) not found` | Use overflow menu on mobile |
| `reader-annotations-and-admin.spec.ts:29` | Bookmarks | `element(s) not found` | Use overflow menu on mobile |
| `reader-annotations-and-admin.spec.ts:38` | Export Notes | `element(s) not found` | Use overflow menu on mobile |

**SW registration PAGE ERRORs** (`workbox-lc5HeaeP.js:2:2838`) are non-fatal
workbox lifecycle warnings in preview mode. They appear as console errors but
do NOT cause test failures. Can be suppressed with `page.on('pageerror')` filter.

- Root cause confirmed: container query layout hides desktop buttons on mobile
- Create a helper function in `apps/tests/fixtures.ts`:
  ```ts
  async function openToolbarAction(page, actionName) {
    const isMobile = page.viewportSize().width < 768;
    if (isMobile) {
      await page.getByRole('button', { name: 'More Options' }).click();
      await page.getByRole('button', { name: actionName }).click();
    } else {
      await page.getByRole('button', { name: actionName }).click();
    }
  }
  ```
