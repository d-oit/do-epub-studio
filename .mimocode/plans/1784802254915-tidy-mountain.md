## P0: EPUB Parser Timeouts (TIER-1 Requirement)

**AGENTS.md rule**: "MUST apply parser timeouts to EPUB content parsing to prevent infinite loops on malformed input."

**Current state**: No timeout mechanism exists on the EPUB parsing pipeline. Bounded regex guards (ADR-034) protect against ReDoS but NOT against:
- DOMPurify processing hanging on deeply nested/malformed HTML
- OPF/XML parsing hanging on malformed XML
- Archive extraction (fflate) processing malformed ZIP archives
- Rendition detection on edge cases

### Implementation Plan

#### Phase 1: Core Timeout Utility

| File | Action | Complexity |
|------|--------|------------|
| `packages/shared/src/errors.ts` | Add `TimeoutError` class extending `AppError` | Low |
| `packages/shared/src/timeout.ts` | **Create** ‚Äî `withTimeout()`, `checkDeadline()`, `createDeadline()` | Medium |
| `packages/shared/src/index.ts` | Add export for timeout module | Trivial |

**Design decisions**:
- `withTimeout<T>(fn, options)` ‚Äî async wrapper using AbortController for cancellable operations
- `checkDeadline(deadline, operation, timeoutMs, traceId)` ‚Äî synchronous `performance.now()` check for sync code paths
- `sanitizeEpubDocument` must remain **synchronous** (epubjs content hooks don't await callbacks) ‚Äî use `checkDeadline()` between passes, NOT `withTimeout()` Promise wrapper
- `checkDeadline` should check every N iterations in TreeWalker loops (not every iteration) to avoid ~5% overhead

#### Phase 2: Integration Points (parallelizable)

| File | Default Timeout | Approach |
|------|----------------|----------|
| `packages/reader-core/src/archive-validator.ts` | 10s | `withTimeout` wrapper + `signal.aborted` checks in `onfile` callback |
| `packages/reader-core/src/epub-parser.worker.ts` | 10s | Import `withTimeout` from shared (worker already uses bundler) |
| `packages/reader-core/src/sanitizer.ts` | 5s | Synchronous `checkDeadline()` between DOMPurify passes + every 100 TreeWalker elements |
| `packages/reader-core/src/fixed-layout.ts` | 2s | `checkDeadline()` between regex extractions |
