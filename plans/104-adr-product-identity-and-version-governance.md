## Context

ADR-102 established `apps/web/src/config/app-identity.json` as the single
source for the **web app** display name and the root `VERSION` file as the
single source for the **shipped version**. That decision is correct but
scoped only to the web runtime. Two problems remain repo-wide:

### 1. The product name is not spelled consistently

The canonical brand string in `app-identity.json` is **`d.o.EPUB Studio`**
(short name `d.o.EPUB`). A repo scan on 2026-06-21 found at least five
divergent spellings outside the web runtime:

| Spelling | Locations (representative) |
|---|---|
| `d.o.EPUB Studio` (canonical) | `apps/web/src/config/app-identity.json`, `LoginPage.test.tsx` |
| `d.o. ePUB Studio` | `README.md:1` (title) |
| `do EPUB Studio` | `packages/ui/src/AppLogo.tsx` (`aria-label`), `components.test.tsx`, `docs/coding-guide.md` (incl. `VITE_APP_NAME=do EPUB Studio`) |
| `EPUB Studio` | `packages/ui/src/__stories__/Header.stories.tsx`, `apps/worker/src/routes/admin/auth.ts:112` (email subject), `apps/web/README.md`, `apps/worker/README.md`, `packages/testkit/README.md`, `docs/setup-local.md`, `docs/reading-insights.md` |
| `do EPUB Studio` | stale build artifact `apps/web/dist/manifest.json` |

There is no written rule defining the canonical string, its allowed
short/ASCII forms, or where each may appear. This produces accessibility
drift (the logo `aria-label` and the visible name disagree), inconsistent
emails, and documentation that contradicts the running UI.

### 2. The version source has drifted from the changelog

`VERSION` and every `package.json` read `0.1.0`, but `CHANGELOG.md`
already records a released `## [0.1.1] - 2026-06-14` plus a populated
`## [Unreleased]` section. The runtime therefore advertises `0.1.0`
while `0.1.1` shipped. `docs/release-process.md` says the changelog is
auto-generated by release-drafter and must never be edited, yet the
changelog has been hand-edited and `VERSION` was never bumped â€” the
governance described in the doc is not the governance in practice.

## Decision

### Naming convention

1. **Canonical name:** `d.o.EPUB Studio`. **Canonical short name:**
   `d.o.EPUB`. These exact strings live only in
   `apps/web/src/config/app-identity.json` and are imported at runtime
   (per ADR-102). No other source file may hard-code a divergent spelling.
2. **ASCII / slug form:** `do-epub-studio` (already the repo, package
   scope `@do-epub-studio/*`, and GitHub org/repo). Unchanged.
3. **Allowed contextual forms** â€” only these, and only where the literal
   string is unavoidable (docs prose, email copy, Storybook fixtures):
   - Full brand in prose and titles: `d.o.EPUB Studio`.
   - Admin surface qualifier: `d.o.EPUB Studio Admin`.
   - Never: `d.o. ePUB Studio`, `do EPUB Studio`, bare `EPUB Studio`.
   The lowercase-e `ePUB` and the space-separated `d.o. ` forms are
   **disallowed**.
4. **Accessibility strings** (`aria-label`, alt text) MUST use the
   canonical name: `d.o.EPUB Studio logo`.
5. **Generic noun usage is fine.** "an EPUB file", "the EPUB reader",
   "EPUB rendering" describe the format, not the product, and are not
   governed by this ADR.

### Version governance

6. **`VERSION` (repo root) is the single numeric source of truth.** It is
   SemVer, derived from Conventional Commits (ADR-035b / release-process).
7. **All `package.json` `version` fields and `VERSION` MUST match** at all
   times and are bumped together by the `release-management` skill â€” never
   by hand, never independently.
8. **`VERSION` MUST be â‰Ą the highest released version in `CHANGELOG.md`.**
   A released changelog heading without a matching `VERSION` bump is a
   release defect and a blocking gap.
9. **One release mechanism.** Releases are cut via the `release-management`
   skill only (AGENTS.md Tier 2 rule 9). `docs/release-process.md` is
   reconciled with this: the changelog `[Unreleased]` â†’ version section
   promotion and the `VERSION`/`package.json` bump happen in the same
   release commit produced by the skill.
10. **Runtime exposure.** The web app displays `APP_VERSION` from
    `VERSION` (ADR-102). The Worker SHOULD expose the same version on a
    health/diagnostic response so deployed version is observable.

### Enforcement

11. A repo guard (lint script, e.g. `scripts/check-app-identity.mjs`,
    wired into the quality gate) MUST fail on any disallowed spelling in
    tracked source/docs and on a `VERSION` â†” `package.json` â†” changelog
    mismatch. Build artifacts (`apps/web/dist/**`) are excluded.

## Consequences

### Positive

- One spelling everywhere; accessibility labels match the visible brand.
- Version drift becomes a CI failure instead of a silent runtime lie.
- Renaming or version bumps remain single-edit operations (ADR-102 intact).

### Negative

- Adds one lint script and an allow-list it must be kept in sync with.
- Existing files need a one-time normalization pass (tracked in the GOAP).

### Neutral

- The slug `do-epub-studio` and package scope are unaffected.
