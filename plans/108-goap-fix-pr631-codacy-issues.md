## Resolution

**Commit:** `1b3fa45` — `fix(worker): replace any types in telemetry test for Codacy compliance`

### Fix Details

The initial approach used `as unknown as Type` double assertions, but this triggered `@typescript-eslint/no-unnecessary-type-assertion` (the lint rule saw through the `unknown` intermediate). The final solution uses **type guard assertion functions** which satisfy both rules:

1. **Type guards** (`assertOk`, `assertError`) — validate response shape at runtime AND narrow the type for TypeScript, avoiding `any` and unnecessary assertions
2. **`vi.mocked(console.log)`** — replaces `(console.log as any)` with Vitest's typed mock utility
3. **`call[0] as string`** — explicit cast on the specific array element (not the whole array)
