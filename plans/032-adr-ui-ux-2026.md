# ADR-032: UI/UX Standards for 2026 Modern Web

**Status:** Accepted
**Date:** 2026-05-16
**Driven by:** UI/UX Improvement Report 2026

---

## Context

The current UI/UX foundation is solid but lacks several modern 2026 features:
- Perceptually uniform color spaces (OKLCH)
- Wide-gamut P3 color support
- Native View Transitions
- Intelligent, scroll-aware UI components
- Coordinated panel management

## Decisions

### 1. Color System: OKLCH & P3
- Move away from sRGB/Hex to `oklch()` for all design tokens.
- OKLCH provides consistent perceived lightness across different hues, making dark mode and accessibility contrast management much easier.
- Support wide-gamut P3 displays via conditional CSS overrides to provide more vibrant colors where supported.

### 2. Layout & Motion: Scroll-Aware & View Transitions
- Implement the "App Shell" pattern for smooth initial loads and auth resolution.
- Use the View Transitions API (v7) for seamless page-to-page navigation.
- UI elements (toolbars) should be context-aware, hiding on scroll to maximize content area.

### 3. Panel Management
- Enforce mutual exclusivity for side panels. Only one panel (TOC, Settings, etc.) can be active at a time to prevent UI clutter and Z-index conflicts.

### 4. Accessibility & Micro-interactions
- Use `framer-motion` for all state transitions, including micro-animations for floating toolbars.
- Ensure all icon-only actions have proper tooltips and accessible labels.

## Consequences

### Positive
- Future-proof design system compatible with 2026 hardware and browser standards.
- Improved reading focus through intelligent UI behavior.
- Better brand perception through high-fidelity motion and color.

### Negative
- OKLCH might require fallback strategies for very old browsers (though baseline-supported in 2026).
- Increased complexity in state management for coordinated panels.
